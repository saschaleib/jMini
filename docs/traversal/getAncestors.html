<HTMLElement>.getAncestors(…)

Get an array of the Element‘s ancestors, starting from the parent, up to the <html> element, optionally filtered by a callback function.

Usage:

<Array> = <HTMLElement>.getAncestors(callback <Function> = undefined)
Member of:
<HTMLElement>, any valid HTMLElement.
Parameters:
  1. callback – a function that returns true, if an item should be added (<Function>, optional).
Returns:
An <Array> of <HTMLElement>s.

Notes:

  1. The resulting list will only contain <HTMLElement>s, i.e. it will not contain the documentElement. This is an important difference to the parentNode property.
  2. The last item in the resulting Array is always the <HTML> element, unless it is called on the <HTML> element itself, in which case this function returns an empty Array.

Examples:

A simple example could look like this:

document.getElementById('foo') .getAncestors() .forEach( (item) => { … });

By using a filtering function, e.g. to only collect ancestors which are <LI> elements:

let list = document.getElementById('foo') .getAncestors( (element) => { return (element.nodeName == 'LI'); });

Note that the .nodeName property always contains the element name in uppercase letters!
